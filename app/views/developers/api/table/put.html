Update a table meta information

Updates the attributes of a table:

  • name
  • tags
  • privacy: allowed values are "PUBLIC" and "PRIVATE"

Also, this method can be used to update the geometry columns of the table using the attributes:

  • latitude_column and longitude_column if your table is georeferenced by a latitude column and a longitude column
  • or address_column if your table is georeferenced by an address. This atribute allows upto three columns, separated by commas. In this case, a new column is created with the content of the three columns separated by commas

URL

/v1/tables/<identifier>

METHOD

PUT

PARAMETERS

Required

  • identifier: the name of the table (in the URL of the request)
  • the new values for the attributes

EXAMPLE


      PUT
        /v1/tables/species_occurrences
          {
            "name": "species_occurrences_preprocessed",
            "tags": "species,biodiversity,preprocessed_data",
            "privacy": "PRIVATE"
          }

JSON

      {
        "id": "1",
        "name": "species_occurrences_preprocessed",
        "tags": "species,biodiversity,preprocessed_data",
        "privacy": "PRIVATE",
        "schema": [["cartodb_id","number"],["name","string"],["description","string"],["latitude","number"],["longitude","number"],["created_at","date"], ["updated_at","date"]]
      }

EXAMPLE


      # Setting an aggregated address column
      # Our table "check_ins" has the columns address, region and country 
      # and we want to georeference the table using those three columns
  
      PUT
        /v1/tables/check_ins
          {
            "address_column": "address,region,country"
          }

JSON

      {
        "id": "1",
        "name": "check_ins",
        "tags": "...",
        "privacy": "...",
        "schema": [["cartodb_id","number"],["name","string"],["address","string"],["region","string"],["country","string"],["aggregated_address","string","address_column"],["created_at","date"], ["updated_at","date"]]
      }
