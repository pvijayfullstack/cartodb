Authentication

CartoDB implements two different authentication methods:

  • OAuth 1.0
  • API key limited to a domain

The reason to implement the second method is to allow JSONP requests in a easy way.

OAuth

CartoDB implements OAuth v1.0 protocol. To start using it you need to get a token and a secret. Negotiation URLs are the following:

  • https://api.cartodb.com/oauth/request_token
  • https://api.cartodb.com/oauth/authorize
  • https://api.cartodb.com/oauth/access_token

Once obtained a valid access token you can keep it and use many times. For the moment, access tokens don't expire, but we may change this politic in the next weeks.

There are lots of libraries that helps you to get authenticated via OAuth, take a look at this list.

API key

API key method consists in signing every request with a parameter named api_key which is associated with the referrer domain of the request. If you share this api_key anybody can make requests as if were you, so be careful with your code.

This method of authentication is very useful if you plan to perfom JSONP requests to the API.
