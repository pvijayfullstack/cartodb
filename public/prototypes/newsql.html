it's just SQL

The SQL API for CartoDB exposes the full query power of SQL to the CartoDB API user. It has been engineered for
secure, high speed query response and also for highly concurrent access.

  • 1. Authentication

    You must be authenticated to interact with your cartodb data. Take a look to how to do it.

  • 2. Create a request

    This is an example of a typical request you can do throw our API.

    http://user.cartodb.com/api?token=”you_token”&sql=”SELECT * FROM table_name”&rows_per_page=”100”&page=”0”

    • Here is where your oauth token goes in your requests. (Required)

    • Your url encoded SQL statement (Required)

    • Only if you want to paginate. Rows per page and page you want.

  • 3. Get your results

Authentication

API access is secured by SSL and oAuth. OAuth is an authentication protocol that allows users to approve application to act on their behalf without sharing their password. More information can be found at oauth.net or in the excellent Beginner’s Guide to OAuth from Hueniverse.

For secure access to your application you will need to generate an API key on your API Keys page.
